01 / Acerca de mí

Combino criterio visual, análisis funcional y documentación para mejorar productos digitales.

Estoy construyendo mi carrera en Quality Assurance después de una trayectoria en diseño gráfico y proyectos digitales. Esa experiencia me ayuda a observar interfaces con atención al detalle, detectar inconsistencias visuales y entender cómo debería comportarse un producto desde la perspectiva del usuario.

Actualmente me enfoco en pruebas funcionales web y móviles, diseño de casos de prueba, validación de APIs con Postman y documentación de defectos en JIRA. Busco oportunidades como QA Tester Jr., QA Manual Jr. o QA Engineer Trainee, donde pueda aportar orden, criterio, comunicación clara y disposición para seguir aprendiendo dentro de un equipo de calidad.

Proyecto 01

API Testing · Automatización

Urban Grocers API: pruebas automatizadas con Python y Pytest

Problema
Validar que la API permitiera crear kits de productos de forma correcta y que respondiera adecuadamente ante datos válidos, inválidos y valores límite.
Qué hice
Diseñé escenarios de prueba para el campo name, ejecuté pruebas automatizadas con Python y Pytest, trabajé con solicitudes HTTP, JSON y autenticación, y verifiqué respuestas del backend según los requisitos.
Resultado o aprendizaje
Fortalecí mis bases en API Testing, automatización inicial, análisis de reglas de negocio y validación de respuestas del servidor.
  • Python
  • Pytest
  • API REST
  • JSON
  • HTTP
  • GitHub

Proyecto 02

QA Manual · Web · Móvil · API

Urban Routes / Urban Grocers: pruebas funcionales web, móvil y API

Problema
Evaluar funcionalidades clave de aplicaciones web, móviles y servicios backend, asegurando que los flujos principales cumplieran con los requisitos definidos.
Qué hice
Analicé requisitos, diseñé casos de prueba, ejecuté pruebas manuales, validé respuestas de API con Postman y documenté defectos en JIRA con pasos de reproducción, resultado esperado, resultado actual y evidencia.
Resultado o aprendizaje
Desarrollé un proceso más estructurado para detectar, priorizar y comunicar errores de forma clara, útil y reproducible para el equipo de desarrollo.
