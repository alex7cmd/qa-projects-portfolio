01 / Acerca de mí

Combino criterio visual, análisis funcional y documentación para mejorar productos digitales.

Estoy construyendo mi carrera en Quality Assurance después de una trayectoria en diseño gráfico y proyectos digitales. Esa experiencia me ayuda a observar interfaces con atención al detalle, detectar inconsistencias visuales y entender cómo debería comportarse un producto desde la perspectiva del usuario.

Actualmente me enfoco en pruebas funcionales web y móviles, diseño de casos de prueba, validación de APIs con Postman y documentación de defectos en Jira. Busco oportunidades como QA Tester Jr., QA Manual Jr. o QA Engineer Trainee, donde pueda aportar orden, criterio, comunicación clara y disposición para seguir aprendiendo dentro de un equipo de calidad.

Proyecto 01

QA Manual · Functional Testing · Regression Testing

Urban Routes — Pruebas funcionales y de regresión

Problema
Validar que los flujos principales de Urban Routes funcionaran según los requisitos: direcciones, tarifas, licencia de conducir, método de pago, opciones del pedido y mensajes de la interfaz.
Qué hice
Convertí requisitos funcionales en casos de prueba y ejecuté pruebas manuales, UI Testing y regresión. Registré resultados en checklists y documenté bugs en Jira con entorno, severidad, pasos y evidencia.
Resultado o aprendizaje
Fortalecí mi capacidad para validar flujos de usuario, detectar inconsistencias funcionales y visuales, y comunicar errores de forma clara, reproducible y útil para desarrollo.

Proyecto 02

API Testing · Postman · REST API

Urban Grocers API — Pruebas de API con Postman

Problema
Validar que los endpoints de Urban Grocers respondieran correctamente ante datos válidos, inválidos y valores límite, considerando métodos HTTP, parámetros, body JSON, status codes y mensajes de error.
Qué hice
Ejecuté requests en Postman y validé status codes, respuestas JSON y reglas de negocio. Diseñé pruebas positivas y negativas con clases de equivalencia y valores límite, y documenté errores en Jira.
Resultado o aprendizaje
Fortalecí mis bases en API Testing, interpretación de respuestas del servidor, validación de reglas de negocio y documentación de errores técnicos con información suficiente para reproducirlos.
