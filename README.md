# QA Projects Portfolio

Portafolio personal de Alejandro Fernández para presentar experiencia práctica como QA Tester Jr. La landing está construida con HTML, CSS y JavaScript sin frameworks para facilitar el aprendizaje y su publicación en GitHub Pages.

## Estructura

```text
.
├── assets/
│   └── images/          # Foto, capturas y recursos visuales
├── css/
│   └── styles.css       # Variables, layout y diseño responsive
├── js/
│   └── main.js          # Menú, animaciones y formulario
└── index.html           # Contenido y estructura semántica
```

## Ver el proyecto localmente

Abre `index.html` en el navegador o ejecuta un servidor local desde VS Code. Por ejemplo:

```bash
python3 -m http.server 8000
```

Después visita `http://localhost:8000`.

## Recursos por reemplazar

- `assets/images/profile-placeholder.svg`: sustituir por `profile.jpg` y actualizar la ruta en `index.html`.
- `assets/images/project-urban-grocers.svg`: sustituir por una captura limpia de Pytest o Postman.
- `assets/images/project-quality-testing.svg`: sustituir por un collage o captura principal del proceso QA.
- Agregar la URL pública exacta de LinkedIn en la sección de contacto.

## Publicación

El repositorio está preparado para GitHub Pages. En GitHub abre **Settings > Pages**, selecciona **Deploy from a branch**, usa la rama `main` y la carpeta `/ (root)`.
